Word Builder: A Weekend Project

How I spent my weekend building a small web app to demonstrate the exact tech stack you're looking for

What This Actually Is

When I saw your job posting mentioning PHP, JavaScript, and educational applications, I thought since my portfolio site is a little anemic at the moment I'll build something that relates to this work.

Full transparency: This isn't a massive production app—it's a focused demonstration of the specific skills you mentioned in your job listing. I built it in a couple of days specifically to show how I approach the kind of work you're doing at Across Cultures.

Word Builder is a simple English learning game where students build words from letter tiles. Teachers can create accounts, add their students, and track progress. It's straightforward, but it covers all the technical bases you mentioned in your requirements.

What I Built (And How)

PHP Backend

  • Simple class-based structure for managing teachers and students
  • Basic API endpoints for the game functionality
  • Progress tracking that stores to the database

MySQL Database

  • Simple tables for teachers, students, and game results
  • Setup script so you can actually run this locally

JavaScript Frontend

  • Drag-and-drop letter tiles
  • Works on phones and tablets
  • Shows progress in real-time
  • Some fun sound effects because why not?

Educational Approach

  • Made it colourful and engaging for young learners
  • Different difficulty levels (3-letter to 5-letter words)
  • Teacher dashboard to see how students are doing
  • Tried to make it actually useful for EAL learners

Why I Built This for You

Your job posting was really specific about needing PHP/MySQL backend experience with JavaScript frontend skills, particularly for educational applications.

Technical Bits That Match Your Requirements

  • PHP/MySQL: Built the whole backend from scratch - teacher management, student tracking, results storage (exactly the kind of work you mentioned)
  • JavaScript: Interactive game mechanics that actually feel responsive and fun
  • Educational focus: Designed specifically for English language learning, with EAL pedagogy in mind
  • Modular architecture: Organised into clean classes that could easily slot into a larger codebase like yours

How I Approached the Development

  • Started with the database structure (always do the data model first!)
  • Built separate PHP classes for different concerns - makes it easier to work on teams
  • Created a proper API structure so frontend and backend stay cleanly separated
  • Made the JavaScript modular too - UIManager, AudioManager, ProgressTracker classes
  • Added comprehensive error handling because real applications need to handle edge cases

What This Shows About How I Work

  • I think about the end users: Made it work well for both teachers and students, considering the EAL learning context
  • I finish projects: It's not just a proof-of-concept - it actually works end-to-end with proper error handling
  • I plan for the long term: The modular structure means it could easily be extended or integrated into a larger platform

The Educational Side

Since you're specifically working on EAL applications, I tried to think about what actually helps English language learners (and researched a bit of EAL pedagogy while building this):

  • Visual word building (seeing letters combine into words) - supports phonemic awareness
  • Immediate, gentle feedback when they get words right or wrong
  • CVC word patterns (cat, dog, pen) which are fundamental for new English learners
  • Progress tracking that identifies error patterns, not just success rates
  • Different difficulty levels that progress logically through language acquisition stages
  • Touch-friendly interface because many EAL learners are using tablets
